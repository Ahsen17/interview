# 面试与简历准备知识库

这个仓库用于持续维护个人简历材料、目标岗位分析、网络面经、技术复习题库和模拟面试记录。核心目标是把“岗位要求 -> 简历投递 -> 知识复习 -> 模拟面试 -> 评估改进”串成一套可迭代的准备流程。

当前仓库同时包含一个最小 Python 项目，但主要内容是 Markdown 文档。

## 目录说明

```text
.
├── 个人简历/      # 个人基础简历、工作经历、项目经历、定向投递简历
├── 岗位要求/      # 公司/职位/岗位介绍.md 与分析后的岗位要求.md
├── 网络面经/      # 公司岗位面经或通用岗位面经
├── 知识题库/      # 可持续扩展的技术知识点、面试 QA、复习路径
├── 面试模拟/      # 按公司岗位生成的模拟面试题与评估
└── CLAUDE.md / AGENTS.md  # Codex/Agent 使用本仓库时必须遵守的规则
```

每个主要目录下都有自己的 `README.md`，新增材料前建议先阅读对应目录说明。

## 推荐使用流程

1. 在 `岗位要求/公司/职位/` 下放入目标岗位的 `岗位介绍.md`。
2. 让 Agent 基于 `岗位介绍.md` 生成或更新 `岗位要求.md`，提炼真实招聘需求、技术关键词、隐含面试信号和高概率面试主题。
3. 在 `个人简历/` 中维护基础简历、`工作经历.md` 和 `项目经历.md`。
4. 让 Agent 基于目标岗位生成 `个人简历/职位-公司-简历.md`，用于直接投递。
5. 将收集到的面经放入 `网络面经/`，明确公司岗位的使用 `公司-职位.md`，通用面经使用 `职位.md`。
6. 让 Agent 综合 `岗位要求/`、`网络面经/`、`个人简历/` 更新 `知识题库/`。
7. 复习到一定阶段后，要求 Agent 从 `岗位要求/` 列出可选公司岗位，并在选择后生成 `面试模拟/职位-公司-yyyyMMdd-HHmmss.md`。
8. 完成模拟题后，让 Agent 基于你的回答生成评估，补充参考答案、得分点、不足和复习建议。

针对某公司-岗位生成模拟面试题时，考察或 QA 范围不得超过该岗位目录下 `岗位要求.md` 的具体要求。`网络面经/`、`个人简历/` 和 `知识题库/` 只能用于补充题目表达、依据材料、追问深度和评分细则，不能把该岗位没有要求的能力扩展为模拟面试题。

## 命名规则

- 岗位介绍：`岗位要求/公司/职位/岗位介绍.md`
- 岗位分析：`岗位要求/公司/职位/岗位要求.md`
- 定向简历：`个人简历/职位-公司-简历.md`
- 明确公司岗位面经：`网络面经/公司-职位.md`
- 通用岗位面经：`网络面经/职位.md`
- 模拟面试题：`面试模拟/职位-公司-yyyyMMdd-HHmmss.md`
- 模拟面试评估：可更新原模拟题文件，也可另存为 `面试模拟/职位-公司-yyyyMMdd-HHmmss-评估.md`

Markdown 文件名不要使用数字前缀排序；知识题库的阅读顺序统一维护在 `知识题库/知识题库阅读索引.md`。

## Agent 使用约定

使用 Codex 或其他 Agent 处理本仓库时，应优先遵守：

1. 根目录 `AGENTS.md`。
2. 相关目录的 `README.md`。
3. Agent skills 规则。

Agent 生成内容时必须基于本地材料，不得虚构个人经历、项目指标、技术栈或面试来源。若某个增强表述有价值但缺少证据，应标记为 `待用户确认`。

生成公司-岗位模拟面试题时，每道题都应能回指到目标岗位的 `岗位要求.md` 中的具体要求、加分项、隐含面试信号或高概率面试主题，不得超纲。
