# 面试模拟目录说明

本目录用于保存面向具体公司和岗位的模拟面试题，以及完成答题后的评估结果。

## 使用流程

1. 请求 Agent 开始模拟面试。
2. Agent 先从 `岗位要求/` 中列出已有的公司-岗位选项。
3. 选择目标公司和岗位。
4. Agent 基于所选岗位的 `岗位要求.md` 确定考察范围，并结合 `网络面经/`、`个人简历/`、`知识题库/` 生成模拟面试题。
5. 你在题目文件中的“我的回答”下作答。
6. 完成后请求 Agent 评估，Agent 再补充参考答案、评分、得分点、不足和复习建议。

## 文件命名

模拟面试题：

```text
职位-公司-yyyyMMdd-HHmmss.md
```

示例：

```text
后端开发工程师-XX 公司-20260705-203000.md
```

评估文件可以直接更新原文件，也可以另存为：

```text
职位-公司-yyyyMMdd-HHmmss-评估.md
```

## 初版模拟题模板

初版模拟题不提供参考答案，只提供 Q 侧信息。

针对具体公司-岗位的模拟题，考察范围不得超过该岗位 `岗位要求.md`。其他目录材料只能作为题目依据、表达参考、追问深度或评分细则的补充，不能引入该岗位要求之外的 QA。

```markdown
# 职位-公司-模拟面试题

- 生成时间：
- 目标公司：
- 目标岗位：
- 依据材料：
- 轮次设计：
- 使用说明：请直接在每道题的“我的回答”下作答；初版不提供参考答案，评估阶段再补充。

## Q1. 问题标题

- 问题：
- 来源或参考：
- 对应岗位要求：
- 考察点：
- 预期回答深度：
- 追问方向：
- 评分维度：
- 我的回答：
```

## 推荐轮次

1. 简历深挖。
2. 项目架构与技术取舍。
3. AI Agent、RAG、LLM 应用工程。
4. 后端工程与系统设计。
5. 数据、评估与成本治理。
6. 行为面与协作复盘。

## 评估内容

评估阶段应补充：

- 参考答案。
- 用户回答评分。
- 得分点。
- 不足之处。
- 技术纠错。
- 更优表达。
- 建议复习材料。

所有模拟题必须有依据或来源，来源可以是本项目中除 `面试模拟/` 外的任意已有文档。针对公司-岗位模拟面试，每道题还必须能回指到所选岗位 `岗位要求.md` 中的具体要求、加分项、隐含面试信号或高概率面试主题。不得凭空编造题目，也不得超出该岗位要求范围。
