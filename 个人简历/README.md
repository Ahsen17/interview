# 个人简历目录说明

本目录用于维护个人背景材料和面向不同公司、不同岗位的定向投递简历。

## 当前用途

- 保存工作经历和项目经历的详细材料。
- 为不同目标岗位生成可直接投递的简历版本。
- 为岗位匹配分析、项目追问、模拟面试评估提供个人经历证据。

## 推荐文件

```text
个人简历/
├── 工作经历.md
└── 项目经历.md
```

`工作经历.md` 和 `项目经历.md` 是后续所有定向简历、项目话术和模拟面试评估的核心依据。请尽量补充真实职责、技术方案、业务背景、指标结果、难点、取舍和复盘。

## 定向简历命名

面向具体公司岗位生成的简历使用：

```text
职位-公司-简历.md
```

示例：

```text
后端开发工程师-XX公司-简历.md
```

## 使用方式

1. 先维护基础材料：`工作经历.md`、`项目经历.md`。
2. 在 `岗位要求/公司/职位/` 中准备好目标岗位的 `岗位介绍.md` 和 `岗位要求.md`。
3. 请求 Agent 生成定向简历，例如：“基于XX公司后端开发工程师岗位要求，生成可投递简历”。
4. Agent 应只使用本目录中已验证的事实。缺少证据但可能有价值的内容必须标记为 `待用户确认`。

## 写作建议

- 优先写清楚“我负责什么、解决什么问题、用了什么方案、结果如何”。
- 技术栈不要只堆关键词，要能对应项目中的真实使用场景。
- 指标、规模、性能提升、成本下降等内容必须真实可解释。
