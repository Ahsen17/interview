# 知识题库目录说明

本目录是持续迭代的复习知识库，用来承接岗位要求、网络面经、个人项目和模拟面试评估中暴露出的知识点。

## 目录结构

```text
知识题库/
├── 知识点/
├── 面试QA/
├── 复习路径/
└── 知识题库阅读索引.md
```

## 子目录用途

`知识点/`：系统复习材料，强调概念、原理、实现、取舍、失败场景和项目落地。

`面试QA/`：面试刷题材料，强调高频问题、考察点、参考回答、追问方向和常见失分点。

`复习路径/`：阅读顺序、P0/P1/P2 优先级、面经问题映射和阶段性冲刺计划。

`知识题库阅读索引.md`：本目录的总入口。阅读顺序写在这里，不通过文件名前缀排序。

## 使用方式

1. 系统复习时，先阅读 `知识题库阅读索引.md`。
2. 需要补齐技术体系时，进入 `知识点/`。
3. 面试前快速刷题时，进入 `面试QA/`。
4. 从面经反查知识点时，优先看 `复习路径/面经问题映射.md`。
5. 模拟面试评估后，将暴露出的薄弱点补回对应知识点或 QA 文档。

## 扩展规则

- 新增系统知识放到 `知识点/` 下最接近的主题目录。
- 新增面试题、追问和回答话术放到 `面试QA/` 下对应主题目录。
- 新增阅读顺序、冲刺安排和映射关系放到 `复习路径/`。
- 不要使用数字前缀命名 Markdown 文件。
- 同一问题跨多个领域时，只在最主要的 QA 文件写完整答案，其他位置通过索引或映射引用。

## 内容来源

知识题库可以来自：

- `岗位要求/` 中提炼出的硬性要求和隐含面试信号。
- `网络面经/` 中反复出现的问题。
- `个人简历/` 中项目经历需要支撑的技术点。
- `面试模拟/` 评估中发现的薄弱项。

可以直接要求 Agent 从以上信息来源，生成一个初版的知识题库，然后根据个人实际情况不断优化迭代。新增内容应尽量说明来源，方便后续追溯为什么要复习这个知识点。
