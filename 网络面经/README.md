# 网络面经目录说明

本目录用于保存收集到的面试经验、面试题、追问记录和候选人复盘。

## 命名规则

明确指向某个公司和岗位的面经：

```text
公司-职位.md
```

示例：

```text
XX 公司-后端开发工程师.md
```

不指向具体公司、只面向通用岗位的面经：

```text
职位.md
```

示例：

```text
后端开发工程师.md
```

## 推荐内容结构

```markdown
# 公司-职位面经

## 来源信息

- 来源：
- 时间：
- 轮次：
- 候选人背景：
- 可信度：

## 高频问题

## 技术追问

## 项目追问

## 行为面问题
```

## 使用方式

1. 收集到面经后，按命名规则放入本目录。
2. 如果面经内容很杂，先保留原文要点，再让 Agent 整理为结构化问题。
3. 请求 Agent 将高频问题映射到 `知识题库/面试QA/` 和 `知识题库/复习路径/面经问题映射.md`。
4. 生成模拟面试时，Agent 应把本目录作为问题风格和追问深度的重要依据。

## 维护规则

- 尽量记录来源、时间和可信度。
- 不要把未经确认的面经当作唯一依据；应结合 `岗位要求/` 和 `个人简历/` 使用。
- 面经中的问题可以抽象为知识题库内容，但不要在多个文件里重复维护完整答案。
- 若内容来自网络，整理时应以摘要和结构化归纳为主，避免大段复制。
