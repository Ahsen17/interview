# 岗位要求目录说明

本目录用于保存目标公司的岗位介绍和从岗位介绍中提炼出的真实岗位要求。

## 目录结构

每个岗位按以下结构组织：

```text
岗位要求/
└── 公司/
    └── 职位/
        ├── 岗位介绍.md
        └── 岗位要求.md
```

示例：

```text
岗位要求/
└── XX 公司/
    └── 后端开发工程师/
        ├── 岗位介绍.md
        └── 岗位要求.md
```

## 文件说明

`岗位介绍.md`：原始岗位信息，可以来自招聘网站、内推 JD、HR 描述或用户整理内容。

`岗位要求.md`：由 Agent 分析后生成，重点不是复述 JD，而是提炼：

- 岗位定位。
- 真实招聘需求。
- 必备能力。
- 加分能力。
- 隐含面试信号。
- 技术关键词。
- 可映射的简历证据。
- 高概率面试主题。
- 待确认信息。

## 使用方式

1. 新增公司和职位目录。
2. 将原始 JD 写入 `岗位介绍.md`。
3. 请求 Agent：“请分析这个岗位介绍并生成岗位要求.md”。
4. 后续生成简历、知识题库、模拟面试时，优先读取 `岗位要求.md`。

## 维护规则

- 一个公司下可以有多个职位。
- 一个职位目录只维护一个主岗位版本；如果 JD 差异很大，建议拆成不同职位目录。
- 不要把面经放到这里，面经统一放到 `网络面经/`。
- 不要把个人经历放到这里，个人材料统一放到 `个人简历/`。
- `岗位要求.md` 可以随 JD 更新持续迭代，但应保留清晰结构，方便后续自动读取。
